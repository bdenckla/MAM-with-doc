Pre-vowel Accents in Chabad’s CTR

The Chabad website has an edition of the Hebrew Bible called The Complete Tanach with Rashi (CTR). (See my document, “On the Provenance of Chabad’s CTR.”)

In several cases, CTR declines to avail itself of distinctions that were available in Unicode 2.0, but, curiously, makes these distinctions in nonstandard ways. On a letter with a vowel mark code point (including HOLAM), CTR distinguishes prepositives from their impositive “lookalikes” by the logical order of an accent code point relative to that vowel:

  • The code point TIPEHA codes both deḥi (!) and tarḥa / tipeḥa.
    • Before a vowel, it codes .
    • After a vowel (the normal order), it codes (the normal meaning of TIPEHA).
  • The code point GERESH codes both geresh muqdam (!) and geresh.
    • Before a vowel, it codes .
    • After a vowel (the normal order), it codes (the normal meaning of GERESH).
  • The code point YETIV codes both yetiv and mahapakh (!).
    • Before a vowel, it codes (the normal meaning of YETIV).
    • After a vowel (the normal order), it codes .

There are many levels of strangeness here.

  • It is not standard to ever code an accent before a vowel, and indeed, such an ordering will be normalized away in some environments, notably, in most web browsers.
  • It is certainly not standard to make a distinction between an accent coded before a vowel and after a vowel.
  • The use of YETIV breaks the pattern established by TIPEHA and GERESH, where the impositive code point does double duty, and the prepositive code point is not used. If the pattern established by TIPEHA and GERESH were to be followed, MAHAPAKH before a vowel would code yetiv (surprising but at least following the pattern) and MAHAPAKH after a vowel would code mahapakh.

In some but not all cases, the logical order of these code points reflects a desired visual ordering. Even when it does reflect a desired visual ordering, few if any fonts will render the marks in the desired visual order. In detail:

  • The code point TIPEHA before a vowel codes deḥi (!), e.g. to encode the segol and deḥi under the letter he in הֶ֭חרשתי in Psalm 32:3 (fully: הֶ֭חֱרַשְׁתִּי ),
    • Instead of ‹POINT SEGOL, DEHI›, CTR uses ‹TIPEHA, POINT SEGOL›.
    • Schematically (using a plus sign to indicate a kind of concatenation), instead of הֶ+ה֭, CTR uses ה֖+הֶ. (The plus sign expressions are meant to be read right to left.)
    • This is very unlikely to have the desired appearance in most fonts. Plus, in normalizing contexts like most web browsers, the font won’t even get a chance to try. E.g. in this document’s context, it will look like this: הֶ֖חרשתי.
    • Above we have ignored a gaʿya that CTR also has under the he. We have ignored it because (1) it is not relevant to the issue at hand and (2) it is one of many “extra” gaʿya marks that CTR has compared to many other editions.
  • The code point TIPEHA after a vowel codes tarḥa / tipeḥa, e.g. Psalm 32:2 וְאֵ֖ין, as is normally the case.
  • The code point GERESH before a vowel codes geresh muqdam (!), e.g. to encode the tsere and geresh muqdam on the letter alef in אֵ֝ליו in Psalm 32:6 (fully: אֵ֝לָ֗יו ),
    • Instead of ‹TSERE, GERESH MUQDAM›, CTR uses ‹GERESH, TSERE›.
    • Schematically (using a plus sign to indicate a kind of concatenation), instead of אֵ+א֝, CTR uses א֜+אֵ. (The plus sign expressions are meant to be read right to left.)
    • This is very unlikely to have quite the desired appearance in most fonts. E.g. in this document’s context, it will look like this: אֵ֜ליו. Because the two marks in question are not both below-marks, this looks pretty close to the desired appearance. But it is still not quite right.
  • The code point GERESH after a vowel codes geresh, e.g. ???
  • The code point YETIV before a vowel codes yetiv, e.g. ???
  • The code point YETIV after a vowel codes mahapakh (!), e.g. to encode the qamats and mahapakh under the letter tav in אתָּ֤ה in Psalm 32:7 (fully: אַתָּ֤ה׀ ),
    • Instead of ‹QAMATS, MAHAPAKH›, CTR uses ‹QAMATS, YETIV›.
    • Schematically (using a plus sign to indicate a kind of concatenation), instead of תָּ+תּ֤, CTR uses תָּ+תּ֚. (The plus sign expressions are meant to be read right to left.)
    • This is very unlikely to have the desired appearance in most fonts. Plus, in normalizing contexts like most web browsers, the font won’t even get a chance to try. E.g. in this document’s context, it will look like this: אתָּ֚ה.

As noted above, CTR’s strange vowel-relative distinctions apply not only to the below-vowels but also to the one above-vowel, HOLAM. E.g., TIPEHA before HOLAM codes deḥi, e.g. in אֹ֖זֶן, אֹ֖מֶר, and כָּל־רֹ֖אַי (Psalm 18:45, 19:4, and 22:8). But, consistent with the general sloppiness of CTR, sometimes TIPEHA appears after HOLAM, even when a deḥi is (or should be) intended, e.g. in בֹּ֖קֶר, and עֹ֖ז in (Psalm 5:4, and 22:11).

Note that TIPEHA before HOLAM (the only above-vowel) is somewhat analogous to GERESH before a below-vowel. In both cases, the ordering does not reflect a desired visual ordering, since in each case, one of the marks is a below-mark and the other is an above-mark. Rather, the ordering reflects at most a desired visual alignment (right-biased rather than centered) of the accent relative to a vowel-free area of its letter. (That area being the letter’s top for GERESH and bottom for TIPEHA). Because this is completely nonstandard, the desired visual alignment is very unlikely to be achieved in most or all fonts.

Just as TIPEHA before HOLAM codes deḥi, so GERESH before HOLAM codes geresh muqdam, e.g. in מִכָּל־רֹ֜דְפַ֗י, פֹּ֜רֵ֗ק, and כָּל־אֹ֜יְבָ֗יו (Psalm 7:2, 7:3, and 18:1). Note that GERESH before HOLAM, like GERESH before a below-vowel, does not reflect a desired distinction in visual ordering, since both geresh and geresh muqdam should, visually, appear before ḥolam. Rather, the ordering reflects at most a desired distinction in visual alignment (right-biased rather than centered) of the accent relative to its letter. Or, if you like, you can think of GERESH logically before HOLAM as coding a geresh visually far before ḥolam, as opposed to GERESH logically after HOLAM, which codes a geresh still visually before ḥolam, but not so far before it.

One might naturally wonder how, on a letter without a vowel, CTR represents the six accents of these three “lookalike” pairs.

  • TIPEHA again codes both deḥi and tarḥa: XXX ambiguously?!
  • GERESH again codes both geresh muqdam and geresh: but these accents are exclusive to the poetic and prose systems respectively, so there is no ambiguity.
  • YETIV again codes both yetiv and mahapakh, but, with no pattern I can discern, sometimes MAHAPAKH codes mahapakh, as in Psalm 32:11 שמח֤ו.
